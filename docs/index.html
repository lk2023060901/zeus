Zeus C++项目文档

欢迎使用Zeus项目！ 这是一个基于C++17开发的大型MMORPG游戏框架，支持分布式部署，采用模块化架构设计。

项目概览

Zeus是一个现代化的C++游戏框架，专为大型MMORPG开发而设计。框架采用分布式架构，支持高并发、低延迟的游戏服务，提供完整的游戏基础设施，包括网络通信、数据存储、日志系统等核心模块。

🚀 分布式架构

支持分布式部署的游戏服务器架构，提供高可用性、水平扩展能力，满足大型MMORPG的性能需求。

🔧 模块化设计

游戏框架采用模块化设计，包含网络、数据库、日志、配置等独立模块，支持灵活的功能组合和扩展。

🧪 完善测试

提供完整的测试体系，包括单元测试、集成测试、压力测试等，确保游戏框架的稳定性和性能。

📦 灵活部署

支持容器化部署、微服务架构，提供灵活的配置选项，适应不同规模的游戏服务需求。

快速开始

开始使用Zeus游戏框架非常简单，只需要几个步骤：

基本构建步骤

# 创建构建目录
mkdir build && cd build

# 配置游戏框架（静态库模式）
cmake .. -DBUILD_SHARED_LIBS=OFF -DBUILD_TESTS=ON

# 编译框架
make

# 运行基础测试
./bin/zeus_test_basic

查看详细构建指南 了解技术栈

技术栈

核心技术

  • C++17 - 现代C++标准
  • CMake 3.16+ - 跨平台构建系统
  • 模块化架构 - 独立可维护的组件设计

第三方库

  • spdlog 1.15.3 - 高性能日志系统
  • nlohmann/json 3.12.0 - 配置和数据处理
  • Boost 1.88.0 - 网络和系统库支持

项目结构

zeus/
├── CMakeLists.txt          # 主框架配置
├── include/common/spdlog/   # 日志模块头文件
├── src/common/             # 基础模块源代码
│   ├── CMakeLists.txt      # 基础模块配置
│   └── spdlog/            # 日志模块实现
├── tests/                  # 测试模块
│   ├── CMakeLists.txt      # 测试管理
│   ├── basic/             # 基础功能测试
│   ├── spdlog_basic/      # 日志模块测试
│   ├── spdlog_performance/ # 性能基准测试
│   ├── spdlog_config/     # 配置系统测试
│   └── spdlog_multithread/ # 并发测试
├── docs/                  # 框架文档
├── cmake/modules/         # 构建模块
└── thirdparty/           # 第三方依赖

主要特性

特性 描述 配置选项
静态/动态库 支持静态库和动态库两种编译模式 BUILD_SHARED_LIBS=ON/OFF
测试控制 可选择性编译测试用例 BUILD_TESTS=ON/OFF
日志系统 高性能日志记录，支持分布式日志收集 配置文件控制
并发处理 高并发游戏逻辑处理，支持多线程安全 框架内置
