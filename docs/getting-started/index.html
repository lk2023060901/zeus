快速开始

欢迎使用Zeus游戏框架！本指南将帮助您快速了解框架并开始游戏开发。Zeus是一个基于C++17开发的大型MMORPG游戏框架，支持分布式部署，采用模块化架构设计。

项目概览

Zeus游戏框架特点: 分布式架构、高并发处理、模块化设计、完整的游戏基础设施

🚀 5分钟快速上手

从下载到运行，只需要5分钟即可搭建基本Zeus游戏框架环境。

mkdir build && cd build
cmake .. && make
./bin/zeus_test_basic

📋 前置要求

  • C++17兼容编译器
  • CMake 3.16+
  • 2GB+ RAM
  • 1GB+ 磁盘空间

快速设置

步骤1: 检查环境

确保您的系统满足基本要求：

# 检查编译器版本
gcc --version    # 需要 GCC 8+
clang --version  # 需要 Clang 6+

# 检查CMake版本
cmake --version  # 需要 3.16+

步骤2: 获取项目

# 如果使用Git
git clone <repository-url> zeus
cd zeus

# 或下载并解压源码包
unzip zeus-project.zip && cd zeus

步骤3: 构建项目

# 创建构建目录
mkdir build && cd build

# 配置项目 (默认静态库模式)
cmake ..

# 编译项目
make -j$(nproc)  # Linux/macOS
# 或
cmake --build . --parallel  # 跨平台方式

步骤4: 运行测试

# 运行基础测试
cd bin
./zeus_test_basic

# 查看生成的日志
ls -la logs/
成功运行输出示例: 
[2025-08-12 10:08:49.845] [game] [info] Zeus游戏框架基础测试 - 日志系统正常工作
[2025-08-12 10:08:49.845] [game] [warning] 这是一个警告消息
[2025-08-12 10:08:49.845] [game] [error] 这是一个错误消息
Zeus框架基础测试完成 - 检查logs目录下的日志文件

CMake构建选项详解

CMake默认配置: 默认不编译tests工程 (BUILD_TESTS=OFF)，使用动态库 (BUILD_SHARED_LIBS=ON)

核心构建参数

参数 默认值 可选值 说明
BUILD_TESTS OFF ON / OFF 控制是否编译测试程序
BUILD_SHARED_LIBS ON ON / OFF 控制生成静态库还是动态库
CMAKE_BUILD_TYPE Release Debug / Release / RelWithDebInfo / MinSizeRel 构建类型和优化级别

常用构建组合

1. 生产模式 (默认)

# 默认配置: 动态库，不编译测试
cmake ..
# 等同于:
cmake .. -DBUILD_TESTS=OFF -DBUILD_SHARED_LIBS=ON
  • ✅ 不编译测试程序，编译速度快
  • ✅ 生成动态库，节省磁盘空间
  • ✅ 适合生产部署和集成开发

2. 开发测试模式

# 开发模式: 包含测试程序
cmake .. -DBUILD_TESTS=ON
  • ✅ 编译所有测试程序，方便功能验证
  • ✅ 保持动态库默认设置
  • ✅ 适合本地开发和调试

3. 静态库部署模式

# 静态库: 部署简单，无运行时依赖
cmake .. -DBUILD_SHARED_LIBS=OFF -DCMAKE_BUILD_TYPE=Release
  • ✅ 生成静态库，部署简单
  • ✅ 无运行时库依赖
  • ✅ Release优化，性能最佳

4. 调试模式

# 调试模式: 包含调试信息，包含测试
cmake .. -DCMAKE_BUILD_TYPE=Debug -DBUILD_TESTS=ON
  • ✅ 包含完整的调试符号
  • ✅ 包含测试程序便于调试
  • ⚠️ 性能较低，仅用于开发

构建结果说明

默认构建 (BUILD_TESTS=OFF, BUILD_SHARED_LIBS=ON)

build/
└── lib/
    ├── libcommon_spdlog.so     # Linux 动态库
    ├── libcommon_spdlog.dylib  # macOS 动态库
    └── libcommon_spdlog.dll    # Windows 动态库

开发测试构建 (BUILD_TESTS=ON, BUILD_SHARED_LIBS=ON)

build/
├── lib/
│   ├── libcommon_spdlog.so     # Linux 动态库
│   ├── libcommon_spdlog.dylib  # macOS 动态库
│   └── libcommon_spdlog.dll    # Windows 动态库
└── bin/                        # 测试程序目录
    ├── zeus_test_basic         # 框架基础测试
    ├── test_zeus_log          # 日志模块测试
    ├── test_spdlog_performance # 性能基准测试
    ├── test_spdlog_config     # 配置系统测试
    ├── test_spdlog_multithread # 并发处理测试
    └── log_config.json        # 日志配置文件

静态库构建 (BUILD_SHARED_LIBS=OFF)

build/
└── lib/
    └── libcommon_spdlog.a      # 静态库文件
注意: 动态库模式下，运行程序时需要确保系统能找到.so/.dylib/.dll文件。

完整构建流程示例

基本开发流程

# 1. 创建构建目录
mkdir build && cd build

# 2. 配置项目 (使用默认设置)
cmake ..

# 3. 查看配置信息
# CMake会输出类似信息:
# -- === Zeus Project Configuration ===
# -- Build type: 
# -- Library type: STATIC
# -- C++ Standard: 17
# -- Build Tests: ON
# -- ===================================

# 4. 编译项目
make -j$(nproc)  # Linux/macOS
# 或
cmake --build . --parallel  # 跨平台

# 5. 运行测试
cd bin
./zeus_test_basic

生产环境构建

# 生产环境优化构建
mkdir release_build && cd release_build

cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_TESTS=OFF \
  -DBUILD_SHARED_LIBS=OFF

make -j$(nproc)

# 结果: 只生成 lib/libcommon_spdlog.a

CMake编译命令对照表

需求 CMake配置命令 适用场景
开发测试 cmake .. 日常开发，功能测试
仅构建库 cmake .. -DBUILD_TESTS=OFF 生产部署，集成开发
动态库开发 cmake .. -DBUILD_SHARED_LIBS=ON 快速编译，内存优化
调试模式 cmake .. -DCMAKE_BUILD_TYPE=Debug 问题诊断，性能分析
性能优化 cmake .. -DCMAKE_BUILD_TYPE=Release -DBUILD_TESTS=OFF 生产环境，最佳性能

快速参考: CMake参数组合

推荐组合:
  • 日常开发: cmake .. (默认)
  • 生产部署: cmake .. -DBUILD_TESTS=OFF -DCMAKE_BUILD_TYPE=Release
  • 快速调试: cmake .. -DCMAKE_BUILD_TYPE=Debug

基本使用

简单日志记录

#include "common/spdlog/zeus_log_manager.h"

int main() {
    using namespace common::spdlog;
    
    // 初始化游戏框架日志系统
    if (!ZeusLogManager::Instance().Initialize("log_config.json")) {
        std::cerr << "游戏框架初始化失败" << std::endl;
        return 1;
    }
    
    // 获取游戏日志器
    auto logger = ZeusLogManager::Instance().GetLogger("game");
    
    // 记录游戏事件日志
    logger->info("游戏服务器启动");
    logger->warn("玩家连接超时警告");
    logger->error("数据库连接失败");
    
    return 0;
}

测试程序说明

测试程序 测试内容 用途

常见问题

编译失败

问题: 编译器不支持C++17特性

解决方案: 升级编译器到完整支持C++17的版本

  • Linux: sudo apt-get install gcc-8 g++-8 或更新版本
  • macOS: 更新Xcode Command Line Tools（需要Clang 6+）
  • Windows: 安装Visual Studio 2017 15.7或更新版本

运行时找不到配置文件

问题: Failed to open log config file

解决方案: 确保配置文件在正确位置

# 检查配置文件
ls -la bin/log_config.json

# 如果不存在，从源码复制
cp ../tests/spdlog/log_config.json bin/
